生成器/处理器的标题和描述, 以及参数的标题和描述, 都是通过国际化方法实现翻译的.

## 设置主语言

`setLocale(locale: string, fallbackLocale?: string)`

默认语言为中文(zh_CN).

```javascript
import mock from "@erhai_lake/mock-service"

mock.setLocale("en_US")

// 如果想同时设置回退语言可以这样
mock.setLocale("en_US", "zh_CN")
```

## 单独设置回退语言

`setFallbackLocale(fallbackLocale: string)`

如果当前语言不存在翻译, 则会回退到备用语言.

默认备用语言为英文(en_US).

```javascript
import mock from "@erhai_lake/mock-service"

mock.setFallbackLocale("zh_CN")
```

## 获取当前语言

`getLocale(): string`

```javascript
import mock from "@erhai_lake/mock-service"

console.log(mock.getLocale()) // en_US / zh_CN
```

## 获取当前回退语言

`getFallbackLocale(): string`

```javascript
import mock from "@erhai_lake/mock-service"

console.log(mock.getFallbackLocale()) // en_US / zh_CN
```

## 调用翻译

`translate(key: string): string`

通常情况下, 方法返回的生成器/处理器都是翻译后的结果.

但可能部分用户有这方面的需求, 可以通过调用翻译方法来实现.

::: note
mock service 提供的翻译方法, 并不支持占位符功能, 只能获取翻译键对应的翻译字符串.

如果你需要使用占位符功能, 请使用其他国际化库, 如 [i18next](https://www.i18next.com/).

PS: 毕竟本来就是为了这个库专门设计的翻译功能, 用不到占位符等功能.
:::

```javascript
import mock from "@erhai_lake/mock-service"

console.log(mock.translate("category.string.title"))
```

## 获取翻译表

`getTranslateTable(): any`

```javascript
import mock from "@erhai_lake/mock-service"

console.log(mock.getTranslateTable())
```
